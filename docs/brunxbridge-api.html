Overview

Use BrunxBridge when a script needs to run across different server stacks without rewriting framework, inventory, target and UI logic for every customer.

The bridge auto-detects compatible resources by default and can be locked to specific systems in the configuration.

Core features

Framework adapter

Unified player, identifier, job, name and money functions.

Inventory adapter

Unified item checks, add/remove item, carry checks and stash registration.

Target adapter

ox_target, qb-target and DrawText fallback through the same export.

UI adapter

Notifications, context menus, input dialogs, progress circles and text UI through ox_lib.

Callbacks

Client and server callbacks using ox_lib.

Society helper

Common society money support for popular banking and management resources.

Installation

ensure ox_lib
ensure oxmysql
ensure brunx_bridge

Ensure your selected framework, inventory and target resource before scripts that depend on BrunxBridge.

Configuration

Config.Framework = 'auto'
Config.Inventory = 'auto'
Config.Target = 'auto'

Examples

Client notification

exports.brunx_bridge:Notify({
    title = 'Success',
    description = 'Action completed.',
    type = 'success'
})

Server item check

local hasPhone = exports.brunx_bridge:HasItem(source, 'phone', 1)
if not hasPhone then
    TriggerClientEvent('brunx_bridge:client:notify', source, {
        description = 'You need a phone.',
        type = 'error'
    })
    return
end

Target zone

exports.brunx_bridge:AddBoxZone('garage_counter', vec3(215.7, -810.1, 30.7), vec3(1.5, 1.5, 2.0), {
    distance = 2.0,
    options = {
        {
            label = 'Open menu',
            icon = 'fa-solid fa-car',
            groups = { mechanic = 0 },
            onSelect = function()
                exports.brunx_bridge:Notify('Garage opened')
            end
        }
    }
})
